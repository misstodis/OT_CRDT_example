How CRDT (Logoot) works: Each character is stored as { id: LogootID, char } where the ID is a unique floating-point position between 0 and 1 (plus site ID and clock as tie-breakers). The document is a sorted array of these pairs — all clients sort by the same ordering rule, so they automatically converge without any server-side transformation. The server is just a dumb relay — it broadcasts operations without transforming them.
